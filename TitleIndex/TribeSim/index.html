TribeSim: Scalable Simulation Systems for Tribler

Three directions:

  • write 100K simulator probably in C excluding piece level (Tamas)
    • Jan David has already produced a C simulator that might be useful
  • get the python simulator running with > 100 peer traces (Rameez)
    • already Rameez has integrated BuddyCast + BarterCast + ModerationCast and Voting systems into one system but only runs for 100 peers
    • perhaps by running for short periods, stopping and saving and restarting we can get more nodes
  • distribute the python system over multiple machines by pre-processing traces (Dave)
    • work out efficient mechanism for distributing via preprocessing for seperable streams with "sync-point" files
