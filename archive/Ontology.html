Ontology for collections and resources

In order to provide useful descriptions of content, it will be useful with an ontology that limits and controls the descriptive information.

Below are some thoughts that should be modified and/or extended.

Collections should be open for various content, say for example both audio and video (a set of films and their soundtracks), so the content type cannot be a part of the collection. A name and description should be required, and a URL, a thumbnail and user tags should be optional.

Resources (swarms?) should be of a certain type, or just "other". Type also should include "Album" (with artist/name/year/genre or something), as well as "Series" (with Season, Episode, Title) etc. For all resources, "user tags" or keywords should of course also be optional.
